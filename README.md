# 需求引导

> 用5W2H提问法，将模糊想法转化为结构化功能清单

## 设计目的

帮助甲方、领导、现场人员等非技术人员清晰表达需求，通过结构化提问引导思考方向，减少反复沟通，快速锁定核心需求。

比如领导说要做仓库管理系统，不做需求引导直接开干，做完才发现"批次管理没做""多仓库不支持"，每次都是"很简单加一下"，结果改架构返工3次。

## 核心能力

### 1. 需求澄清

通过5W2H提问逐步引导，将模糊想法转化为结构化功能清单。

### 2. 需求过滤

识别并拒绝不合理需求：

| 类型 | 示例 |
|------|------|
| 用户与功能不匹配 | K12平台加打赏功能 |
| 业务逻辑冲突 | 既要公开又要保密 |
| 合规/安全风险 | 未成年人平台涉及支付 |

## 5W2H提问顺序

1. **Why** - 核心动机：你想解决什么问题？
2. **Who** - 目标用户：谁会用这个系统？
3. **When/Where** - 使用场景：什么时间、地点使用？
4. **What** - 核心动作：进来后第一步做什么？
5. **How** - 执行方式：自动还是人工介入？
6. **How much** - 约束条件：优先级、有何限制？

## 输出类型

| 需求清晰度 | 输出 |
|-----------|------|
| 清晰 | 功能清单表（含系统/模块/功能/优先级） |
| 模糊 | 需求调研问题清单（3-5个关键问题） |

## 示例

**场景**：领导说"给我上个AI系统"

**对话**：
> 你：您想解决什么问题呢？是客服接待不过来，还是想降低人工成本？
>
> 领导：主要是晚上咨询没人回，重复问题太多
>
> 你：哪些场景最费人工？电商售后、产品咨询、还是...
>
> 领导：主要是售后占了60%
>
> 你：好的，那核心是减少重复问题的自动回复？

**输出（功能清单表）**：
| 序号 | 系统 | 模块 | 功能 | 描述 | 优先级 |
|:----:|------|------|------|------|:------:|
| 1 | 客服端 | 知识库 | FAQ管理 | 维护常见问题与答案 | P0 |
| 2 | 客户端 | 对话 | 智能问答 | 匹配FAQ自动回复 | P0 |
| 3 | 客服端 | 统计 | 问题分类 | 统计高频问题 | P1 |
