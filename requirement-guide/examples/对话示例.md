# 需求引导示例

## 示例1：需求模糊 → 输出问题清单

**输入**：用户说"我想做个东西"

**执行**：
1. "您想解决什么问题呢？"
2. 用户答："就是管理起来方便点"
3. "方便管理什么呢？是库存、员工、还是订单？"
4. 用户答："员工吧"

**输出**：
> 基于目前的信息，有几个问题需要进一步明确：
> 1. 这个系统是给谁用的？HR、行政主管、还是员工自己？
> 2. 现在员工管理有哪些痛点？打卡、请假、还是档案管理？
> 3. 预计多少人使用？

---

## 示例2：需求明确 → 输出功能清单

**输入**：用户说"我想做个考勤系统，给公司行政用的，主要记录打卡和统计缺勤"

**执行**：
1. "打卡是用什么方式？指纹、刷卡、还是手机？"
2. 用户答："手机定位打卡"
3. "统计缺勤是指什么？迟到、早退、还是请假都算？"
4. 用户答："都算，还要能导出报表"

**输出**：

| 序号 | 系统 | 模块 | 功能 | 描述 | 优先级 | 备注 |
|:----:|------|------|------|------|:------:|------|
| 1 | App | 打卡 | 手机定位打卡 | 员工通过手机定位记录上班时间 | P0 | |
| 2 | App | 打卡 | 考勤统计 | 按日/月统计出勤情况 | P0 | |
| 3 | App | 考勤 | 异常提醒 | 迟到/早退/缺勤自动标记 | P1 | |
| 4 | PC | 报表 | 导出Excel | 导出指定时间段的考勤报表 | P1 | |

## 示例3：识别不合理需求

**输入**：用户说"我想做个K12学习平台，要加学生打赏功能"

**执行**：识别到未成年人+金钱交易风险

**输出**：
> K12平台涉及未成年学生，打赏功能存在合规风险。请确认：
> 1. 这个功能是否有特定业务场景必须？
> 2. 是否有替代方案可以实现相同目的？
