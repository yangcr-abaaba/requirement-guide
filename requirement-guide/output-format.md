# 输出格式定义

## 格式一：功能清单表

适用条件：用户需求清晰，具体功能点已明确

```
| 序号 | 系统 | 模块 | 功能 | 描述 | 优先级 | 备注 |
|:----:|------|------|------|------|:------:|------|
| 1 | PC/App/小程序 | 一级模块 | 具体功能 | 一句话描述 | P0/P1/P2 | 风险备注 |
```

## 格式二：需求调研问题清单

适用条件：用户需求模糊，关键信息不明确

- 列出3-5个关键问题
- 每个问题对应一个核心维度（Who/What/How等）
- 问题要具体、可回答
